## 一、产品概述

### 1.1 产品愿景

NoteFlow 是一款面向知识工作者、学生和创意人群的高级笔记应用。它将美观流畅的编辑体验、灵活强大的文件组织能力与 AI 智能功能融为一体，让用户能够专注于思考，而非工具本身。

### 1.2 核心价值主张

- **双栏 Markdown 编辑** — 左写右看，实时渲染，所见即所得
- **随心所欲的组织方式** — 无限层级文件夹，拖拽重命名
- **本地优先，隐私安全** — 数据存于浏览器本地，无需注册账号
- **AI 是助手，不是噪音** — 按需调用，不干扰创作心流（规划中）

### 1.3 目标用户

| 用户群体 | 核心诉求 |
|---------|---------|
| 知识工作者 | 快速记录想法、整理资料、结构化输出 |
| 学生 | 课堂笔记、知识点整理、Markdown 文档编写 |
| 创意工作者 | 灵感收集、故事结构、素材管理 |
| 开发者 | 代码片段管理、技术文档编写、Markdown 预览 |

### 1.4 当前版本说明

> **v0.1 Demo 已完成**，为纯前端 Web 应用，运行于浏览器，数据保存在 localStorage。
> 本文档以当前实现为基准，用 ✅ 标注已实现功能，🔲 标注规划中功能。

## 二、功能需求

### 2.1 编辑器

#### 2.1.1 编辑模式

| 模式 | 状态 | 说明 |
|------|------|------|
| 双栏模式 | ✅ 已实现 | 左侧 Markdown 源码，右侧实时预览，分隔线可拖拽调整 |
| 专注模式 | 🔲 规划中 | 全屏写作，隐藏侧边栏 |
| 沉浸模式（WYSIWYG） | 🔲 规划中 | 富文本所见即所得，隐藏 Markdown 语法 |
| 纯源码模式 | 🔲 规划中 | 隐藏预览，仅展示编辑区 |

#### 2.1.2 文本格式化

**基础格式（Markdown 原生支持）— ✅ 已实现**
- 标题 H1–H6（`#` 语法）
- 加粗（`**text**`）、斜体（`*text*`）、删除线（`~~text~~`）、行内代码（`` `code` ``）
- 有序列表、无序列表、任务列表（`- [ ] / - [x]`）
- 引用块（`>`）、水平分隔线（`---`）
- 超链接（`[text](url)`）
- 代码块（三重反引号 + 语言名，语法高亮）
- 表格（GFM 语法）

**格式化工具栏** — ✅ 已实现
- 工具栏位于编辑区标题栏下方，双栏共用
- H1 / H2 标题：对当前行自动添加 `#` / `##` 前缀（可切换/取消）
- 加粗（**B**）：选中文字包裹 `**text**`；无选中则插入 `****` 光标居中
- 下划线（U）：选中文字包裹 `<u>text</u>`；无选中则插入标签光标居中
- 荧光笔高亮（Mark）：选中文字包裹 `<mark>text</mark>`，预览区显示黄色底色
- 字体大小调节：下拉菜单选项 12 / 14 / 15 / 16 / 18 / 20 / 24 px，实时更新预览区字号
- 导出 PDF：一键触发浏览器打印，仅输出预览内容（不含侧边栏与工具栏）

**高级格式** — 🔲 规划中
- 数学公式（LaTeX / KaTeX 渲染）
- 可折叠块（Callout / Toggle）
- 脚注与尾注

#### 2.1.3 富媒体嵌入 — 🔲 规划中

- 图片（粘贴、拖入、URL 引用）
- 文件附件
- 网页书签卡片

#### 2.1.4 编辑体验

| 功能 | 状态 |

| 查找与替换 | 🔲 规划中 |
| 版本历史 | 🔲 规划中 |

| 阅读时间估算 | 🔲 规划中 |

### 2.4 AI 功能 — 🔲 规划中（v1.0）

> AI 功能均为**按需触发**，不自动介入写作流程。

#### 2.4.1 写作辅助

| 功能 | 触发方式 | 说明 |
|------|---------|------|
| **续写** | 选中文字 → AI 菜单 | 根据上下文生成后续内容 |

| **摘要** | 全文 → AI 菜单 | 生成摘要，可选长度 |
| **翻译** | 选中文字 → AI 菜单 | 支持中/英/日/韩等主流语言 |

#### 2.4.2 知识管理

- 跨笔记问答（针对知识库提问，AI 综合多篇笔记回答）

- 自动标签建议

#### 2.4.3 AI 设置

- 配置 API Key（支持 Claude / OpenAI）
- 一键关闭所有 AI 功能（隐私优先模式）

## 三、技术实现（当前 Demo）

### 3.1 技术栈

| 层次 | 技术 |
|------|------|
| 框架 | React 19 + Vite 8 |
| 状态管理 | Zustand |
| Markdown 渲染 | react-markdown + remark-gfm |
| 数据存储 | localStorage |
| 样式 | 纯 CSS（CSS 自定义变量） |
| ID 生成 | uuid v4 |

### 3.2 数据模型

```json
{
  "folders": [
    { "id": "uuid", "name": "文件夹名", "parentId": null, "createdAt": 1710000000000 }
  ],
  "notes": [
    { "id": "uuid", "title": "笔记标题", "content": "Markdown 内容", "folderId": null, "createdAt": 1710000000000, "updatedAt": 1710000000000 }
  ]
}
```

### 3.3 项目结构

```
noteflow/
├── src/
│   ├── App.jsx / App.css          # 根组件 + 全局样式/CSS变量
│   ├── main.jsx                   # 入口
│   ├── store/useStore.js          # Zustand 全局状态
│   ├── utils/storage.js           # localStorage 读写工具
│   └── components/
│       ├── Sidebar/               # 侧边栏（文件夹树、操作按钮）
│       ├── Editor/                # 编辑器（分栏、Markdown 预览）
│       └── EmptyState/            # 空状态页面
```

### 3.4 性能表现

| 指标 | 实测值 |
|------|--------|
| 冷启动时间 | < 500ms（Vite 开发模式） |
| 编辑器输入延迟 | < 16ms（60fps） |
| Markdown 渲染 | 实时（随输入同步） |
| localStorage 写入 | 防抖 500ms，无感知 |

## 五、关键用户故事

1. **作为一名学生**，我可以打开浏览器直接开始记笔记，用 Markdown 格式化内容，右侧实时看到渲染效果，无需安装任何软件。

2. **作为一名作家**，我可以创建多个文件夹分别存放不同项目的章节，双击重命名，右键删除，自由整理结构。

3. **作为一名开发者**，我可以用三重反引号写代码块，选择编程语言，右侧预览中看到带语法高亮的代码展示。

4. **作为一名隐私敏感用户**，我的笔记全部存在本地浏览器中，无需注册账户，关闭标签页后数据依然安全保留。

## 七、已知问题与限制

| 问题 | 说明 | 计划修复版本 |
|------|------|------------|
| localStorage 5MB 上限 | 大量笔记或含大段文字时可能触碰上限 | v0.5（IndexedDB 迁移） |
| 无回收站 | 删除笔记/文件夹立即永久删除 | v0.2 |
| 无法导出笔记 | 关闭浏览器无法备份数据 | v0.2 |
| 搜索功能缺失 | 笔记多时难以查找 | v0.2 |
| 仅支持单标签页 | 多标签页操作可能导致数据不同步 | v0.5 |
