# PRD：自律Quest —— 游戏化人生管理系统

---

## 一、项目概述

**项目名称：** 自律Quest（Self-Discipline Quest）

**技术栈：** Vue 3 + Vite + Pinia + Vue Router + LocalStorage

**目标：** 一款以 RPG 游戏化为核心设计语言的个人自律管理工具，通过经验值、技能熟练度、成就奖励等游戏机制激励用户坚持自我提升。

**受众：** GitHub 面试官 / 自我管理需求的用户

---

## 二、核心功能模块

### 模块 1：个人主页（Profile）

**功能描述：**
用户进入应用后首先看到个人主页，类似 RPG 角色卡片。

**具体需求：**

| 字段 | 说明 |
|------|------|
| 头像 | 支持上传本地图片或选择预设像素风头像 |
| 昵称 | 可编辑文本 |
| 座右铭 | 一行简短格言 |
| 总经验值 | 汇总所有任务获得的 XP，显示当前等级 |
| 等级称号 | 根据总 XP 自动计算（如：新手冒险者 / 熟练旅者 / 精英探索者） |
| 技能熟练度总览 | 以进度条形式展示所有技能的当前熟练度 |
| 目标列表 | 用户可添加 1-5 个长期目标（纯文本，带完成状态） |

**UI 风格：** 像素风 / RPG 角色卡，暗色背景 + 金色/绿色高亮

---

### 模块 2：打卡日历（Check-in Calendar）

**功能描述：**
展示当月 31 天的日历，支持每日打卡记录与可视化。

**具体需求：**

- 默认展示**当前月份**的完整日历（7列网格布局）
- 每个日期格子有两种状态：
  - **未打卡**：灰色空格
  - **已打卡**：发光绿色 ✓ 标记（带 CSS 动画）
- 点击某一天 → 弹出确认框 → 确认后标记为打卡成功
- **清空按钮**：一键清空当月所有打卡记录（二次确认弹窗防误操作）
- 顶部显示当月打卡统计：`已打卡 X 天 / 当月 XX 天`
- 连续打卡天数统计，显示"当前连击：X 天 🔥"
- 数据持久化至 LocalStorage，刷新不丢失

**可视化增强：**
- 打卡格子支持 hover 光效
- 打卡成功时触发粒子动画或小型庆祝特效

---

### 模块 3：任务系统（Quest Board）

**功能描述：**
用户自定义任务，完成后获得经验值并提升关联技能熟练度。

**任务数据结构：**

```
任务名称     | 经验值(XP) | 关联技能 | 累计完成次数 | 操作
学习 Vue     |    5 XP    | 编程     |    12 次    | [+1完成] [编辑] [删除]
跑步 5km     |    4 XP    | 体能     |    8 次     | [+1完成] [编辑] [删除]
```

**具体需求：**

**创建任务：**
- 任务名称（必填，自由输入）
- 经验值设置：下拉选择 3 / 4 / 5 XP
- 关联技能：从已有技能中选择，或输入新技能名称自动创建
- 完成次数：初始为 0，自动记录

**记录完成：**
- 点击 `[+1完成]` 按钮 → 立即：
  1. 该任务完成次数 +1
  2. 关联技能熟练度 + 对应 XP 值
  3. 用户总 XP + 对应 XP 值
  4. 触发小型升级动画提示（如：`+5 XP ⚡`）

**任务管理：**
- 支持编辑任务名称、XP、关联技能
- 支持删除任务（二次确认）
- 任务列表按技能分组展示（可切换为列表视图）

---

### 模块 4：技能熟练度系统（Skill Mastery）

**功能描述：**
每个技能拥有独立的熟练度进度条，达到里程碑时触发奖励设置。

**技能数据结构：**

```
技能名称：编程
当前熟练度：145 / 200
进度条：[===============>         ] 72.5%
下一奖励：满 200 时 🎁
```

**里程碑与奖励机制：**

| 里程碑 | 触发条件 | 用户操作 |
|--------|---------|---------|
| 初级成就 | 熟练度达到 100 | 弹出设置奖品弹窗 |
| 中级成就 | 熟练度达到 200 | 弹出设置奖品弹窗 |
| 高级成就 | 熟练度达到 500 | 弹出设置奖品弹窗（扩展） |

**奖励设置弹窗：**
- 用户输入自定义奖品文字（如："买一杯奶茶 🧋" / "游戏时间 +2小时"）
- 保存后在技能卡片上显示"已解锁奖励：XXX 🎉"
- 历史已解锁奖励可在技能详情页查看

**技能管理：**
- 支持添加新技能（名称 + 可选图标）
- 支持删除技能（删除前提示：已有任务关联此技能）
- 技能页面以卡片网格展示

---

### 模块 5：番茄钟（Pomodoro Timer）

**功能描述：**
内置番茄工作法计时器，完成一个番茄后可关联任务记录完成。

**具体需求：**
- 默认：专注 25 分钟 + 休息 5 分钟
- 支持自定义时长
- 计时结束时浏览器通知提醒
- 完成一个番茄 → 弹出选择框："本次番茄关联哪个任务？" → 一键记录完成 +XP
- 当日番茄数统计显示

## 三、数据持久化方案

所有数据存储在 **LocalStorage**，无需后端：

```
localStorage keys:
- sq_profile        # 个人资料
- sq_checkins       # 打卡记录 {year-month: [day1, day2...]}
- sq_tasks          # 任务列表
- sq_skills         # 技能列表及熟练度
- sq_rewards        # 里程碑奖励配置
```

## 四、页面路由设计

```
/           → 个人主页 (Profile)
/calendar   → 打卡日历 (Check-in)
/quests     → 任务看板 (Quest Board)
/skills     → 技能熟练度 (Skills)
/timer      → 番茄钟 (Pomodoro)
```

侧边栏导航，带图标 + 文字，移动端折叠为底部 Tab Bar。

## 五、UI 设计规范

| 项目 | 规范 |
|------|------|
| 主色调 | 深色背景 `#1a1a2e` + 紫金配色 `#e94560` / `#f5a623` |
| 字体 | 标题用像素风字体（Press Start 2P），正文用 system-ui |

| 响应式 | 支持 PC + 移动端 |

## 六、项目亮点（面向面试官）

1. **Vue 3 Composition API** 全面使用，展示 `setup`、`ref`、`computed`、`watch`
2. **Pinia 状态管理**，跨组件数据共享
3. **Vue Router** 多页面路由
4. **自定义动画**：CSS keyframes + JS 动态控制
5. **LocalStorage 持久化**封装为独立 composable `useStorage`
6. **组件化设计**：技能卡片、任务行、日历格等均抽象为可复用组件
7. **无后端依赖**，一键部署到 GitHub Pages
